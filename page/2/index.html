hexo 博客搭建指南

Install hexo

若npm很慢，请将npm的源换成taobao的。
如果出现问题，可以参见这篇文章：hexo 安装与配置

Initial hexo folder

进入一个空文件夹，执行

Run server

默认打开本地的4000端口，可以用于调试。可以马上看到hexo的hello world页面

More info: Server

Create a new post

新建文章到 source/_posts/xxx.md, 通过相应md编辑器编辑即可。

More info: Writing

Link to github

  • 打开_config.yml, 在最后deploy 字段编写如下：

  • 该github的仓库名，必须命令为 your_github_name.github.io

  • 保证 npm install hexo-deployer-git –save 执行成功；

Generate static files

生成静态文件到public目录，在发布之前必须先进行这个操作

More info: Generating

Deploy to remote sites

将文章部署到远程仓库，比如github。 在_config.yml中“deploy”进行配置。

More info: Deployment

Custom domain

  • 购买域名
  • 进入相应工程，创建名为CNAME的文件，文件内容为域名
  • 工程右上角有个”Settings”, 进入之后,在“custom domain”处填写域名
  • 在域名商那里设置A record, 或者CNAME record。

Support math

  • 执行操作: npm install hexo-math –save

  • 在_config.yml中添加 plugins: hexo-math

    行内公式： $e^{i\pi} + 1 = 0$
    行间公式： $$A = U \Sigma V$$

Tips

  • hexo的命令一般可以用首字母缩写代替，比如 hexo server = hexo s
    hexo deploy –generate = hexo d -g
  • 新建文章，并发布的操作一般是：
    • hexo n “your_article_title”
    • hexo clean
    • hexo d -g
  • 在本地调试，不需要进行generate 和deploy.
